Google Play Games Services Plugin

This plugins uses Google Play Games Services. Which offers games serivces for free. This includes leaderboards, achievements, turn based multiplayer, etc.

Limitations:

This require a native.webview which is only supported by Mac, Android, and iOS at this moment (5/24/2016). You also need a google play developer account (find out more) which is a one time free of 25$

Functions:

gpgs.init(clientId, clientSecret, redirectUri)

clientId

(required)(string) this is the client id at console.developers.google.com under your project

clientSecret

(required)(string) this is the client secret at console.developers.google.com under your project

redirectUri

(required)(string) this is redirect Uri(or Url) you in at console.developers.google.com under your project

gpgs.login(listener)

this will show web pop if their is no refresh token stored

listener

(required)(function) this will let you know if the user login in with event.error or event.isError

gpgs.quickLogin(listener)

great if your want to "try" to login without showing login

listener

(required)(function) this will let you know if user login was successful by returning no event.error or event.isError

gpgs.signOut(listener)

signs out user

listener

(required)(function) this will let you know if user log out was successful by returning no event.error or event.isError

gpgs.getMe(listener)

gets users info

listener

(required)(function) if successful (tell by event.error) will return a table of ther users info (learn more)

gpgs.request(url ,sendType, listener, params, body)

makes a google play game service request, all example with be from getMe function above

url

(required)(string) the url to make request example: "https://www.googleapis.com/games/v1/players/me"

sendType

(required)(string) this is the type http request Should be (in all caps): "PUT", "POST", or "GET" example: getMe uses "GET"

listener

(required)(function) This will returns what you expect from a network.request and event.response usually needs to be json.decoded

params

(optional)(table) use key value to make request, these are usually green tables on google's devs site

example: getMe uses {playerId = "me"}

body

(optional)(table) use key value to make request, these are the purple table on google's devs site. example: the create turn based match(not a build in api for this plugin) body looks like {kind = "games#turnBasedMatchCreateRequest", invitedPlayerIds={"105289036114841020197"}, variant = 1, autoMatchingCriteria= {kind= "games#turnBasedAutoMatchingCriteria", minAutoMatchingPlayers = 2, maxAutoMatchingPlayers = 2, exclusiveBitmask = 0}, requestId = myRequestId}

For more on requests check out google rest docs
